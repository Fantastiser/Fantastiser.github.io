python列表添加字典需要注意地方

##python列表添加变量的存储机制
在写软件工程课设的时候，发现了python列表存入字典的现象

list=[]
dict = {}
for i in range(0,3):
    dict[“a”] = i
    list.append(dict)
print list

输出结果：

[{‘a’: 2}, {‘a’: 2}, {‘a’: 2}]

这是因为列表存入字典的是字典的地址，每次修改字典的值就会引起之前存储的字典的变化。想要避免这个情况，就需要每次再循环的时候都要重新定义字典。

list=[]
for i in range(0,3):
    dict = {}
    dict[“a”] = i
    list.append(dict)
print list

输出结果：

[{‘a’: 0}, {‘a’: 1}, {‘a’: 2}]

如果存入的不是字典而是普通变量就没有这种情况了。

list=[]
a=0
for i in range(0,3):
    a=i
    list.append(a)
print list

输出结果:

[0, 1, 2]

这是因为a=i相当于每次都重新定义了变量，所以列表中存入的变量地址不同。
